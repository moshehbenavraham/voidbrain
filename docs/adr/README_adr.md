# Architecture Decision Records

This directory holds architecture decision records for Voidbrain.

## Purpose

- Capture durable design decisions that affect project structure or behavior.
- Keep each record short, specific, and linked to the relevant code or docs.
- Use one file per decision with an ASCII-only filename such as
  `0001-decision-title.md`.

## Guidance

- Prefer factual, reviewable entries over broad design essays.
- Avoid duplicating the main PRD, architecture overview, or implementation
  notes.
- Link to the code or documentation that motivated the decision when needed.
